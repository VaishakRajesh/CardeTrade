A9. BUSINESS WORKFLOWS & SIGNALS
--------------------------------------------------------------------------------
Status: DONE
Files:  app/signals.py, app/apps.py

Automated signal handlers (post_save):

  1. create_listing_on_verification (Batch)
     - When batch status = 'verified'
     - Reads QualityVerification.verified_price_per_kg
     - Auto-creates fixed-price Listing
     - Updates batch status to 'listed'

  2. notify_seller_on_order (Order)
     - When new order created
     - Creates ORDER_PLACED notification for seller (farmer)

  3. notify_farmer_on_bid (Bid)
     - When new bid placed
     - Creates BID_RECEIVED notification for listing farmer

  4. update_conversation_timestamp (Message)
     - When new message sent
     - Updates Conversation.last_message_at for sorting

Status workflows implemented:

  Batch:    pending → under_review → verified → listed → sold
            pending/under_review → rejected

  Order:    pending → confirmed → processing → shipped → delivered
            any status → cancelled / disputed

  Bid:      active → accepted / rejected / outbid / expired

  Dispute:  open → under_review → resolved → closed

  Payment:  pending → completed / failed / refunded
